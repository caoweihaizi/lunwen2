# 第 5 阶段产出文档：遥测缺失观测生成

## 文档定位

登记 P5 阶段**实际产出**的全部交付物与缺失机制特性，供 P6/P7/P9 使用。所有数值基于 2026-07-08 真实执行（seed=0）。

P5 在 P4 完整 ground truth 副本上注入结构化缺失，生成运行时可见观测。

---

## 1. 代码模块（`src/missing/`）

| 模块 | 关键接口 | 用途 |
|---|---|---|
| `inject.py` | `inject_mcar` / `inject_mar` / `inject_block` | 三类缺失机制（单 shard） |
| `observed.py` | `build_observed_shard` | 观测表 + 信息年龄（跨 shard 接续） |

入口：`src/p5_main.py`（流式逐 shard，Block 用全局事件列表跨 shard）。测试：`test_missing.py`（4 测）。

---

## 2. 数据产出（`data/processed/observed/`，被 `.gitignore` 忽略）

```
data/processed/observed/
├── <policy>_<scenario>_<mechanism>/   # 39 个数据集
│   ├── observed_shard0.npz            # 114 shard
│   └── ...
└── <run>_missing_summary.json         # 9 个 run 的缺失统计
```

每个 observed_shard 含：
- `ls_truth (M,10)`：真值（与 P4 一致，未改）
- `mask (M,3)`：三字段（carried/queue/util）观测掩码，True=观测
- `observed (M,10)`：缺失字段置 NaN
- `age (M,3)`：信息年龄（距上次观测的时隙数，跨 shard 接续）
- `field_cols`：[4,5,7]

机制：mcar20 / mcar40 / mar20 / block10；compound 场景额外 compound_mar。

---

## 3. 缺失机制特性（真实执行结果）

| 机制 | 目标 | 实际 | 验证 |
|---|---|---|---|
| MCAR 20% | 0.20 | 0.200 | ✓ 独立随机 |
| MCAR 40% | 0.40 | 0.400 | ✓ |
| MAR 20% | 0.20 | 0.200 | ✓ 高负载缺失 60% vs 低负载 18%（非退化） |
| Block 10 | ~0.20 | 0.181 | ✓ 连续 10 时隙，跨 shard 事件 |

**MAR 关键验证**：高负载链路（util≥0.8）缺失率 59-60%，低负载（util<0.2）18%——MAR 正确体现"控制面高负载时遥测易丢失"，未退化为 MCAR。这是技术大纲 §4.4.2 的核心要求。

**Block 跨 shard**：预生成全局 (链路, 起始时隙) 事件列表，逐 shard 应用，保证连续 10 时隙缺失正确跨越 shard 边界。

**真值不变**：P5 ls_truth 与 P4 link_state 逐元素相等（np.array_equal=True），缺失只改观测副本。

---

## 4. 9 run × 机制矩阵

9 run（3 策略 × 3 场景）× 4 机制 = 36 数据集；compound 3 run 额外 +compound_mar = 39 数据集。每 run 56966 时隙 × 114 shard。

缺失率跨 run 一致（因机制参数固定，与策略/场景无关），符合预期。

---

## 5. 给 P6/P7/P9 的接口契约

1. **观测数据**：`np.load('observed/<run>_<mechanism>/observed_shard{N}.npz')`：
   - P7 训练用 `observed`（含 NaN，缺失感知）
   - P9 评价用 `ls_truth`（完整真值）
   - `mask`/`age` 作为模型输入特征
2. **字段**：carried=列4, queue=列5, util=列7
3. **机制**：mcar20/mcar40/mar20/block10，技术大纲 §4.5 的 S2-S5 压力场景对应
4. **跨 shard age**：已接续，age 反映距上次有效观测的真实时隙间隔

**P5 不交付**：模型训练（P7）、校准（P9）、路由（P10）。
