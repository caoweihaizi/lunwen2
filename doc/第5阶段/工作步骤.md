# 第 5 阶段工作步骤：遥测缺失观测生成

## 文档定位

本文档是 P5 阶段的可执行工作步骤，面向 Claude Code 执行。

P5 目标（见 `完整工作流.md` §P5）：在 P4 完整 ground truth 的**副本**上注入结构化缺失，生成运行时可见观测（MCAR/MAR/Block Missing）。对应技术大纲 §4.4 / §4.2 步骤 12。

**P5 的核心原则（技术大纲 §4.2 / §4.6）：**
- 缺失**只在完整 ground truth 的副本上注入**，不改变 ground truth 本身；
- ground truth 用于离线评价，运行时观测用于预测器/路由训练；
- 缺失只改观测，不改真值；
- 同一完整样本的不同缺失版本不得跨训练/测试集。

**P5 的边界：**
- ✅ 做：在 P4 link_state 真值上生成缺失掩码 M、信息年龄 Δ、缺失类型、到达时间；产出观测表。
- ❌ 不做：模型训练（P7）、校准（P9）、路由 RL（P10）。P5 只产出"运行时可见的不完整观测"。

## 缺失机制（技术大纲 §4.4）

### MCAR（§4.4.1）
以 20% 和 40% 概率独立删除流量/利用率/队列遥测。建立标准缺失率曲线。

### MAR（§4.4.2）
缺失概率随利用率和队列增大：
\[
P(M_{ij,t}=0)=\sigma(aU_{ij,t}+b\tilde Q_{ij,t}+c)
\]
参数标定使总体缺失率达 20%。模拟控制面高负载时遥测易丢失。

### Block Missing（§4.4.3）
单链路连续缺失 10 个时隙。

## 步骤 1：缺失注入器（WRITE）

### 1.1（WRITE）`src/missing/inject.py`

实现三类缺失机制，均在 link_state 真值副本上操作（输入为 shard 的 `(M,10)` 数组，列索引：carried=4, queue=5, util=7）：

- `inject_mcar(link_state_shard, rate, rng, field_cols=(4,5,7)) -> (mask, observed_shard)`：
  - 对每条记录每个指定字段独立以 `rate` 概率置为缺失（mask=0）；
  - 返回掩码 M 与缺失后的观测 shard（缺失处填 NaN 或保留上次有效值）。

- `inject_mar(link_state_shard, target_rate, rng) -> (mask, observed_shard, params)`：
  - 利用率 U（列7）、归一化队列 Q̃ = queue/buffer（列5）；
  - \(P(\text{missing})=\sigma(aU+b\tilde Q+c)\)；
  - **标定 a,b,c 使总体缺失率≈target_rate（20%）**：用 grid search 或二分 c（因 a,b 可固定比例，c 控制整体水平）；
  - 建议固定 a=2, b=1（利用率影响强于队列），二分 c 使缺失率达 20%。
  - **P4 实际负载注意**：66 星稀疏流量下中位 util=0，仅 P95≈0.45、max=1.0。MAR 在低负载链路缺失概率几乎相同（全靠 c），高负载链路缺失率显著更高——这正是 MAR 的预期行为（控制面高负载时遥测易丢）。标定后须验证"高负载（util>0.8）链路缺失率 vs 低负载链路缺失率"有显著差异，否则 MAR 退化为 MCAR。

- `inject_block(link_state, block_len, rng) -> (mask, observed)`：
  - 随机选链路，该链路连续 block_len（10）个时隙缺失；
  - 可多条链路同时 block，总缺失率约 20%（由选多少链路控制）。

### 1.2（WRITE）信息年龄与到达时间

- `compute_age(mask, times) -> age`：
  - 对每条链路每个字段，age = 距离上次有效观测（mask=1）的时隙数；
  - 永久缺失则 age 持续增长；
  - `arrival_time`：每次 mask 从 0→1 转换的时刻（新观测到达）。

### 1.3（WRITE）`tests/test_missing.py`

断言：
- MCAR 实际缺失率≈目标（±1%）；
- MAR 总缺失率≈20%，且高利用率链路缺失率显著高于低利用率（相关性检查）；
- Block 连续长度=10，单链路连续；
- age 计算：连续缺失 N 时隙后 age=N；
- ground truth 未被修改（缺失只改副本）。

## 步骤 2：观测表生成（WRITE）

### 2.1（WRITE）`src/missing/observed.py`

`build_observed(link_state, missing_mechanism, cfg, rng) -> dict`：
- 输入 P4 link_state 真值 + 缺失机制；
- 输出观测表（技术大纲 §4.3.4 遥测段）：
  ```
  edge_id, time_slot, link_active,
  observed (mask), arrival_time, age, missing_type,
  carried_observed, util_observed, queue_observed,  # 缺失处为 NaN 或上次有效值
  ground_truth_carried, ground_truth_util, ...      # 真值保留供离线评价
  ```
- **关键**：真值列与观测列并存，P7 训练用观测列，P9 评价用真值列。

### 2.2（WRITE）复合缺失场景

技术大纲 §4.5.3 复合压力场景固定为 `20% MAR + 局部4倍突发 + 10%链路故障`。P5 在 compound 场景的 link_state 上叠加 MAR（突发和故障已在 P4 体现）。

## 步骤 3：覆盖所有 run 与缺失机制（WRITE + RUN）

### 3.1（WRITE）`src/p5_main.py`

主流程（**逐 run 逐 shard 流式**，避免内存膨胀）：
1. cfg/seed/log/rec；
2. 遍历 P4 的 9 个 run（3 策略 × 3 场景）；
3. 对每个 run，**逐 shard** 读 `link_state_shard{N}.npz`：
   - 对每个 shard 生成 MCAR 20%、MCAR 40%、MAR 20%、Block 10 四种缺失观测；
   - compound run 额外生成复合（MAR）观测；
   - 立即写观测 shard 到 `data/processed/observed/<policy>_<scenario>_<mechanism>/observed_shard{N}.npz`；
   - 释放该 shard 内存，处理下一个；
4. Block 缺失需跨 shard 边界处理（连续 10 时隙可能跨 shard），处理最后一个/首个时隙时检查相邻 shard；
5. 指标：各机制实际缺失率、平均连续长度、与拥塞状态相关性（高负载 vs 低负载链路缺失率差）；
6. 更新 DATA_DICT；`rec.finish()`。

### 3.2（注意）

- 缺失注入用独立 Generator（`make_seed_stream`），可复现；
- **不得跨 run 复用同一缺失掩码**（每个 run 的链路状态不同，缺失应独立生成）；
- 测试集缺失可更强（技术大纲 §4.6），但 P5 先统一生成，P7/P9 按划分取用；
- **流式**：绝不全量加载 9 run，逐 shard 处理即释放。
