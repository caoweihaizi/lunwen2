### 3.2（WRITE）`src/common/config.py` — 配置加载与哈希

实现：
- `load_config(overrides: dict | None = None) -> OmegaConf`：加载 `configs/config.yaml`，支持命令行 override；
- `config_hash(cfg) -> str`：对配置做稳定序列化（排序键）后取 SHA256 前 12 位，作为实验唯一标识；
- `resolve_paths(cfg) -> dict`：返回各目录的绝对路径，保证跨机器可移植。

## 步骤 4：实现种子管理（WRITE）

### 4.1（WRITE）`src/common/seed.py`

实现：
- `seed_everything(seed: int)`：同时设置 `random` / `numpy` / `torch`（CPU 与 MPS）种子；
- `make_seed_stream(base: int, kind: str) -> Iterator[int]`：基于 `(base, kind)` 用 numpy RandomState 生成可复现的子种子流，供同一阶段内多个随机组件（如数据缺失注入、模型 dropout）独立但不冲突地取种；
- 种子 kind 枚举：`data` / `model` / `routing`，与 `config.yaml` 三条种子链对应。

### 4.2（WRITE）`tests/test_seed.py`

断言：相同 `(base, kind)` 两次调用 `make_seed_stream` 产出相同序列；不同 kind 产出不同序列。

### 4.3（RUN + VERIFY）运行测试


```bash
#!/usr/bin/env bash
set -euo pipefail
```
