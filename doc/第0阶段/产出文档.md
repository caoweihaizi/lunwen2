- `load_config(cli=True)` 用 `OmegaConf.from_cli()` 解析 `run.sh` 透传的 `key=value` 参数（修复了初版 override 不生效的漏洞）。

## 3. 关键接口契约（后续阶段依赖，P0 后冻结）

后续阶段通过以下约定与 P0 基础设施对接，**不得擅自改动**（改动需同步影响下游并更新本文档）：

1. **配置访问**：`from src.common import load_config` 得到 OmegaConf 对象，字段点访问（`cfg.data.timeslot_minutes`）。
2. **种子**：阶段开始 `seed_everything(cfg.seed.<kind>)`，阶段内多随机组件用 `make_seed_stream(cfg.seed.<kind>, kind)` 取子种子。
3. **实验记录**：每阶段 main 创建 `ExperimentRecord`，结束 `.finish()`；指标经 `.log_metric()` 落盘，不另写散文件。
4. **日志**：`get_logger(__name__, stage)`，同时进控制台与 `logs/<stage>.log`。
5. **代码版本**：`ExperimentRecord` 自动记录 `get_code_version()`，无需手动调用。
6. **入口**：每阶段提供 `src/pN_main.py`，由 `scripts/run.sh PN <seed>` 调用。

**实际验证**：以上 6 条接口在 P0 自检中全部调用通过（见审计文档 §1）。
