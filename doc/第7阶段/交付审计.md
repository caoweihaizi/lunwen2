## 2. 调参过程（关键）

**v1 问题**：val_loss 卡在 0.2575，模型坍缩预测 0。

**诊断**：模型 q50 std=0.002，与标签相关 -0.02——完全坍缩。根因是单调头 `q05=softplus(z0)` 强制非负，但标准化标签 median=-0.255 有大量负值，模型无法拟合负值只能坍缩到 0。恒定预测中位数 loss=0.183 < 模型 0.263，证实模型比常数还差。

**修复**：q05 = z0（可负），q50 = q05 + softplus(z1)，q95 = q50 + softplus(z2)。单调性仍保证（softplus 增量≥0）。

**v2 结果**（+ hidden=128/2层GRU/50epoch/LR调度/梯度裁剪）：val_loss 0.1473，改善 44%，低于恒定中位数 0.183。

**遗留**：模型 q50 std 仍小（0.001），时序区分弱（与标签相关 0.07）——模型学到预测中位数但未充分区分高低流量。P8 评价驱动进一步优化。
