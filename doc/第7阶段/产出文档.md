## 2. 模型架构（技术大纲 §5.1-5.4，轻量版）

```
历史窗口 (B, W=30, 12) + 未来计划 (B, 3, 3)
        ↓
GRU-D 缺失恢复（§5.2）：γ=exp(-W·age)，缺失处用 (1-γ)·mean 恢复
        ↓
GRU(hidden=64) 编码时序
        ↓
拼接未来计划 MLP → decoder MLP
        ↓
单调分位数头：y05=softplus(z0), y50=y05+softplus(z1), y95=y50+softplus(z2)
        ↓
输出 (B, n_h=2, 3) [h=1,3 的 q05/q50/q95]
```

**特征（12维）**：carried/util/queue（observed）+ 3 mask + 3 age + delay/dist/active。

**关键简化（诚实记录）**：
- 技术大纲 §5.3 的节点 GAT 需整图消息传递，但 P7 按"单边样本"训练（每样本一条边），GAT 退化为边级 MLP。完整 GAT 在 P10 路由环境（候选邻居集合）实现更自然。
- 路由状态输入（§5.4 下一跳/分流比例）因 P4 keep_detail=False 未存明细，暂用 link_active + 物理属性近似。

## 4. 已知问题（不阻断 P8，待调参）

| 问题 | 原因 | 处置 |
|---|---|---|
| val_loss 卡在 0.2575 不降 | (1) queue 特征全 0（66星稀疏流量队列空）；(2) carried 极右偏，均值预测即低 loss；(3) 模型容量/特征不足 | P8 评价前调参：加容量/学习率调度/特征工程；或在 P8 报告中说明 |
| GAT 退化为边级 MLP | 单边样本无图结构 | P10 路由环境实现完整 GAT |
| 路由状态未输入 | P4 未存明细 | 近似或 P4 重跑 keep_detail |

val_loss 0.2575 在标准化空间（std≈31188 Mbps），绝对误差约 8000 Mbps，相对 mean 7871 偏高——模型基本预测均值，时序学习不足。P8 评价时会用 MAE/RMSE 等原始量纲指标判断。

## 5. 给 P8/P9/P10 的接口

1. **checkpoint**：`torch.load('models/predict/predict_v1.pt')` → state_dict + normalize + config；
2. **加载模型**：`PredictModel(n_feat=12, hidden=64, n_h=2, h_max=3).load_state_dict(...)`；
3. **输入**：历史 W=30 时隙边特征 (12维) + 未来计划 (3,3)；
4. **输出**：(n_h=2, 3) 分位数 [h=1,3 的 q05/q50/q95]；
5. **标准化**：carried/util/queue 用 normalize mean/std（训练集估计）。

**P7 不交付**：CQR 校准（P9）、路由 RL（P10）、实验评价（P8）。P7 产出"冻结的预训练预测器 + 原始分位数"。
