## 物理模型选择（关键设计决策）

技术大纲 §3.1 要求"Python 生成 Walker 星座轨道参数与卫星位置；按轨道位置计算覆盖区域和星间链路可用性"。P2 采用**简化 J2 圆球轨道解析模型**，不引入 sgp4/pyephem 等重依赖：

- **轨道**：圆轨道，J2 摄动只影响升交点赤经漂移（RAAN drift），用于跨轨链路可见性的长期一致性；卫星位置用球面三角解析解（真近点角线性化），精度对 4 个月仿真足够。
- **地球**：球面近似（半径 6371 km），不考虑 J2 以外的非球形摄动、大气阻力、日月引力。
- **时间基准**：以仿真起始时刻 t=0 为参考，所有卫星初始相位按 Walker 几何排布。
- **坐标系**：地心惯性系（ECI）球坐标（RAAN、倾角、纬度幅角），转换到 ECEF 经纬度用于覆盖计算。

**理由**：硕士论文仿真的目的是验证路由/预测方法，不是高精度轨道预报；Walker 规则星座的几何对称性使解析模型完全够用。简化模型可复现、无外部数据依赖、运行快。这一选择的边界在论文 8.2 局限性说明。

## 步骤 1：星座轨道生成器（WRITE）

### 1.1（WRITE）`src/topo/orbit.py`

实现 Walker 星座卫星位置计算：

- `class WalkerConstellation`：从 `cfg.constellation.walker`（planes=6, sats_per_plane=11, inclination=53°, altitude=550km）构造；
- 66 星排布：Walker Delta `6/66/2`（6 面 11 星，面间相位偏移 F=2 颗）——**F 待定**，技术大纲未指定，P2 采用 F=2（Iridium 类常用值），在 config 注明；
- `position(sat_id, t_seconds) -> (lon, lat, alt)`：给定时刻返回卫星地心经纬度（ECEF）；
  - 真近点角 `M = M0 + n*t`（n=√(μ/a³)，圆轨道）；
  - J2 RAAN 漂移：`Ω(t) = Ω0 + Ωdot*t`，`Ωdot = -3/2 * J2 * (Re/a)^2 * n / cos(i)`；
  - 球面三角转地心经纬度；
- `all_positions(t_seconds) -> np.ndarray (66, 3)`：全部卫星 [lon,lat,alt]；
- 物理常数：μ=3.986e5 km³/s²，Re=6371 km，J2=1.0826e-3。

## 步骤 2：计划拓扑与链路可见性（WRITE）

### 2.1（WRITE）`src/topo/topology.py`

实现星间链路可见性与计划拓扑：

- `inter_satellite_link_visible(pos_i, pos_j, alt_km) -> bool`：两星间是否有视距链路（不被地球遮挡）；
  - 几何判据：两卫星位置 + 地心，连线与地球表面不交，或两星连线最近距地心 > Re；
  - 同时考虑最大 ISL 距离阈值（如 5000 km，避免过远链路）；
- `build_planned_topology(positions, cfg) -> dict`：给定某时刻全部卫星位置，构造**四邻居**计划拓扑：
  - 轨内邻居：同轨道面 ±1（永久可见，前后各一）；
  - 跨轨邻居：相邻轨道面，按 Walker 几何选最近且可见的两颗（左右各一）；
  - 返回邻接矩阵 `adj (66,66)` + 每条链路的 `distance_km`、`prop_delay_ms`（距离/光速）；
  - `prop_delay_ms = distance_km / 299.792 * 1000`；
- `link_capacity(cfg) -> float`：链路容量 C_ij（统一值，如 10 Gbps，P3 标定时可调）。

**关键**：轨内链路永久存在（同面卫星相对静止）；跨轨链路随轨道运动可见性变化（接近极轨时跨轨链路可能中断——但 53° 倾角非极轨，跨轨链路通常持续可见，仍需逐时隙判可见性）。技术大纲 §3.1"每颗卫星最多连接两个轨内邻居和两个跨轨邻居"——固定四邻居结构，跨轨邻居可见性变化时切换到次近可见卫星。

## 步骤 3：时间序列拓扑生成（WRITE）

### 3.1（WRITE）`src/topo/topo_series.py`

实现整段仿真时间的拓扑序列：

- `generate_topology_series(cfg, t_start, t_end, dt_seconds) -> dict`：
  - 从 t_start 到 t_end 按 Δt=3min 步长生成每时隙的卫星位置 + 计划拓扑；
  - **时间范围与 P1 数据对齐**：以 GÉANT 的 `S_resampled` 时刻为基准（P3 要把 S(t) 映射到拓扑上），所以 P2 拓扑时间序列应覆盖 P1 重采样的时间范围。读 `data/interim/geant/resampled_times.pkl` 取时刻列表，按相同时刻生成拓扑；
  - 返回：
    ```python
    {
      "times": list[datetime],          # 与 P1 resampled_times 对齐
      "positions": np.ndarray,          # (T, 66, 3) lon/lat/alt
      "adj": np.ndarray,                # (T, 66, 66) bool
      "distances": np.ndarray,          # (T, 66, 66) km，不可见处为 0 或 inf
      "prop_delays": np.ndarray,        # (T, 66, 66) ms
      "sat_ids": list[int],
    }
    ```

**内存**：T=56966（GÉANT 重采样长度）×66×66×bool ≈ 248MB（adj），加 distances/prop_delays float32 各 ~495MB，总约 1.2GB。可接受但偏大；若内存紧张，存为稀疏（每时隙只存存在的边）。**P2 采用稀疏存储**：每时隙存 `edge_list (n_edges, 2)` + `edge_dist (n_edges,)`，遵循全局工程约定。

## 步骤 4：地面覆盖映射（WRITE）

### 4.1（WRITE）`src/topo/coverage.py`

实现卫星对地面区域的覆盖映射：

- `satellite_coverage(sat_pos, ground_lonlat, min_elevation_deg=10) -> bool array`：给定卫星位置和一批地面点（lon,lat），判断哪些点被该卫星覆盖（仰角 ≥ min_elevation）；
  - 几何判据：地面点与卫星的地心角距 ≤ 覆盖角 `θ = arccos(Re*cos(el)/(Re+alt)) - el`；
- `build_coverage_series(positions_series, worldpop_weights, cfg) -> dict`：
  - 对每时隙、每个有效像元（来自 P1 WorldPop 三元组），找覆盖它的卫星（取仰角最大的，多星覆盖时选主服务星）；
  - **稀疏存储**：每时隙存 `pixel_to_sat (K,)`（K=5124万有效像元 → 卫星 id），或更省内存地存 `sat_pixel_buckets`（每颗卫星覆盖的像元索引列表）；
  - 返回覆盖映射 + 每像元的服务卫星 + 仰角。

**内存关键**：K=5124万 × T=56966 时隙的完整覆盖矩阵不可能存（2.9e15 项）。P2 只存**每个时隙的覆盖映射**（K 个 int8/int16 卫星 id = 56966×5124万×2B ≈ 5.8TB，仍不可能）。

**解决方案**：覆盖映射不在 P2 预计算全部时隙，而是：
- (a) 预计算一个**代表性周期**（如 1 个轨道周期 ~32 时隙）的覆盖映射，P3 用周期性假设复用；或
- (b) P2 只存卫星位置序列 + 覆盖判据函数，P3/P4 在线按需计算覆盖（每时隙算一次，K 次判据）。

**P2 采用 (b)**：存位置序列 + 提供 `coverage_at_time(t)` 函数，P3/P4 调用时实时计算。这样 P2 产物小、接口清晰。覆盖映射的实际聚合在 P3 完成。

## 步骤 5：目的势函数预计算（WRITE）

### 5.1（WRITE）`src/topo/potential.py`

实现计划拓扑上的目的势函数（技术大纲 §6.3，P10 action masking 用）：

- `potential_shortest_hops(adj, dst) -> np.ndarray (66,)`：给定目的卫星 d，用 BFS 算每颗卫星到 d 的最短跳数；
- `potential_propagation_delay(prop_delays, dst) -> np.ndarray (66,)`：给定目的 d，用 Dijkstra 算每颗卫星到 d 的最小传播时延；
- `potential_series(adj_series, dst_list) -> dict`：对每个目的卫星、每时隙预计算势函数。
  - 但 66 目的 × 56966 时隙 × 66 值 = 2.5亿 float32 ≈ 1GB，且 P2 不必预计算全部目的（P10 训练时按需算即可）；
  - **P2 只预计算几个代表性目的**（如 6 个均匀分布的目的卫星）的势函数时间序列，作为 P10 接口验证；完整势函数 P10 在线计算。
