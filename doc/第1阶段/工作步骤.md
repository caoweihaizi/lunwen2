# 第 1 阶段工作步骤：外部数据接入与预处理

## 文档定位

本文档是 P1 阶段的可执行工作步骤，面向 Claude Code 执行。每步标注动作类型（CREATE / WRITE / EDIT / RUN / VERIFY），可作为逐条执行清单。

P1 目标（见 `完整工作流.md` §P1）：把三个外部数据集（GÉANT、Abilene、WorldPop）读成统一的、可追溯的内部格式，供 P2–P3 合成外生 OD 需求使用。

**P1 不做**：不合成 OD 需求（P3）、不算卫星覆盖（P2）、不标定总量（P3）、不训练模型。P1 只负责"原始文件 → 干净的内部数据结构 + 元数据字典"。

三个原始文件在项目根目录（被 `.gitignore` 忽略，不入库但本地存在）：

| 数据集 | 文件 | 格式 | 规模 | 单位 |
|---|---|---|---|---|
| GÉANT | `directed-geant-uhlig-15min-over-4months-ALL-native.tgz` | SNDlib native ASCII | 11460 矩阵，15min，22 节点 | Mbps |
| Abilene | `directed-abilene-zhang-5min-over-6months-ALL-native.tgz` | SNDlib native ASCII | 48096 矩阵，5min，12 节点 | Mbps |
| WorldPop | `global_pop_2025_CN_1km_R2025A_v1.tif` | GeoTIFF | 43200×17280，EPSG:4326，1km | 人/像元 |

**SNDlib native 格式结构**（GÉANT 与 Abilene 同构，已确认）：

```
?SNDlib native format; type: network; version: 1.0
# network demandMatrix-<gran>-<timestamp>-<net>

META (
  granularity  = 15min
  time  = 20050504-1630
  unit  = MBITPERSEC
  origin  = Steve Uhlig, http://totem.info.ucl.ac.be/dataset.html
)

NODES (
  at1.at ( 16.3729 48.2091 )      # 节点id (经度 纬度)
  ...                              # GÉANT 22 节点，Abilene 12 节点
)

LINKS ( )                          # 本数据集为空（仅需求矩阵）

DEMANDS (
  at1.at_be1.be ( at1.at be1.be ) 1 26.455799 UNLIMITED
  # demand_id ( src dst ) routing_unit value max_path_length
  ...
)
```

**关键观察：**
- 文件名含时间戳 `demandMatrix-<net>-<gran>-YYYYMMDD-HHMM.txt`，可解析出每个矩阵的时刻；
- NODES 段带经纬度——P1 提取备用（P3 地理映射验证可用），但不做地理投影；
- DEMANDS 是**有向** OD（含反向），GÉANT 22 节点 → 22×21=462 条有向需求；
- 单位 `MBITPERSEC` = Mbps（注意：与 TOTEM 匿名版的 kbps 不同，本数据用 SNDlib/Uhlig 版）。

## 步骤 1：实现 SNDlib native 解析器（WRITE）

### 1.1（WRITE）`src/data/sndlib_parser.py`

实现 `parse_sndlib_native(text: str) -> dict`，解析单个 native 文件文本，返回：

```python
{
  "network": str,            # "geant" 或 "abilene"
  "granularity": str,        # "15min" / "5min"
  "timestamp": str,          # "20050504-1630"
  "unit": str,               # "MBITPERSEC"
  "nodes": [(node_id, lon, lat), ...],   # 保留顺序
  "demands": [(src, dst, value), ...],   # 有向，value 为 float Mbps
}
```

解析要点：
- 跳过首行 `?SNDlib...` 与 `#` 注释行；
- `META (...)` 段：正则提取 `granularity` / `time` / `unit`；
- `NODES (...)` 段：每行 `<id> ( <lon> <lat> )`，用正则 `(\S+)\s*\(\s*(-?[\d.]+)\s+(-?[\d.]+)\s*\)`；
- `LINKS (...)` 段：本数据集为空，跳过；
- `DEMANDS (...)` 段：每行 `<id> ( <src> <dst> ) <ru> <value> UNLIMITED`，提取 src/dst/value（value 转 float）；
- **健壮性**：用正则而非逐字符解析；对空行、多余空格容错；value 为 0 或负值保留原值不剔除（缺失/异常在 P5 处理）。

### 1.2（WRITE）`src/data/sndlib_parser.py` 内置自检

模块末尾加 `if __name__ == '__main__'` 块：用 `tar -xOz` 抽取 GÉANT 第一个文件并解析，打印节点数、demand 数、首末 demand。供步骤 1.4 手动验证。

### 1.3（WRITE）`tests/test_sndlib_parser.py`

- timestamp/unit/granularity 正确；

不依赖真实大文件，保证测试快且独立。

## 步骤 2：批量加载与统一时间索引（WRITE）

### 2.1（WRITE）`src/data/load_traffic.py`

实现 `load_traffic_matrix_archive(cfg, which: str) -> dict`，`which` ∈ `{"geant","abilene"}`：

- 从 `cfg.<which>.file` 取文件名，定位项目根的 tgz；
- 流式遍历 tgz 内所有 `.txt`（不解压到磁盘，用 `tarfile` 内存读取）；
- 对每个文件调用 `parse_sndlib_native`；
- 从文件名或 META.time 解析时刻 → `datetime`；
- **按时间排序**（文件名顺序不一定严格递增，必须排序）；
- 构建 OD 矩阵张量/数组：形状 `(T, N, N)`，`N` = 节点数，`[i,j]` = 节点 i→j 的需求（对角线 0）；
- 节点顺序固定为首个文件的 NODES 顺序（跨矩阵一致），建 `node_id → index` 映射；
- 返回：
  ```python
  {
    "network": str,
    "times": list[datetime],        # 长度 T
    "nodes": list[str],             # 长度 N，有序
    "node_coords": list[(lon,lat)], # 与 nodes 对齐
    "matrix": np.ndarray,           # (T, N, N) float64, Mbps
    "granularity_min": int,
    "unit": str,
  }
  ```

**健壮性要点：**
- 校验每个矩阵节点数一致（不一致则报错，记录跳过的文件到日志）；
- 校验矩阵数 == `cfg.<which>.n_matrices_expected`（GÉANT 11460、Abilene 48096），不符则警告但继续；
- 检测时间缺口：相邻时刻间隔应等于粒度（15min/5min），缺口记录到日志（GÉANT 已知有少量缺口，属正常）；
- 内存：GÉANT 11460×22×22×8B ≈ 44MB，Abilene 48096×12×12×8B ≈ 55MB，可全载入内存。

## 步骤 3：提取时间模式（WRITE）

### 3.1（WRITE）`src/data/time_patterns.py`

实现两个提取函数（对应技术大纲 §4.1.1）：

- `extract_global_intensity(matrix) -> np.ndarray`：返回 `S(t) = matrix.sum(axis=(1,2))`，长度 T，单位 Mbps；
- `extract_od_relative_pattern(matrix) -> np.ndarray`：返回 `p_{od}(t) = Y_{od,t} / S(t)`，形状 (T,N,N)，对 S(t)=0 的时刻置 0（避免除零）。

这两个量是 P3 合成外生 OD 需求的"时间模式源"：
- `S(t)` 决定卫星网**总量**的时间变化；
- `p_{od}(t)` 决定**相对 OD 结构**（仅作参考，P3 空间结构由 WorldPop 重定义）。

## 步骤 4：重采样到决策时隙 Δt=3min（WRITE）

### 4.1（WRITE）`src/data/resample.py`

实现 `resample_to_timeslot(S, times, target_minutes: int) -> (np.ndarray, list[datetime])`：

- 输入：原始 `S(t)` 与对应时刻，原始粒度 15min（GÉANT）/5min（Abilene）；
- 目标：`Δt = cfg.data.timeslot_minutes` = 3 min；
- 方法：以目标时刻为锚点，对原始序列做**线性插值**（`numpy.interp`，按时间戳数值化）；
- 输出：重采样后的 `S(t')` 与目标时刻列表。

**注意：**
- 只重采样 `S(t)` 这个一维总量序列（与 P3 用法一致），**不重采样完整 OD 矩阵**（P3 不直接用 GÉANT 的 OD 结构，只借时间模式，重采样整矩阵浪费且无必要）；
- Abilene 同理；
- 边界处理：目标时刻落在原始时间范围外则不外推（截断到原始范围）。

### 4.2（WRITE）重采样前后功率谱对比（VERIFY）

实现 `compare_spectrum(before, after, out_path)`：对重采样前后 S(t) 画功率谱密度，确认 3min 重采样不引入虚假高频。结果存 `data/interim/diagnostics/resample_spectrum.png`，作为 P6（4.7 数据质量验证）的证据。

### 4.3（VERIFY）

## 步骤 5：WorldPop 栅格读取与区域权重（WRITE）

### 5.1（WRITE）`src/data/worldpop.py`

实现 `load_worldpop(cfg) -> dict`：

- 用 `rasterio` 打开 `cfg.worldpop.file`；
- 读取波段 1（masked=True，自动掩 nodata=-99999）；
- 返回：
  ```python
  {
    "data": np.ndarray,        # (H, W) masked array，人/像元
    "transform": Affine,       # 像元→经纬度
    "crs": str,                # "EPSG:4326"
    "bounds": (left,bottom,right,top),
    "width": int, "height": int,
    "nodata": float,
  }
  ```

### 5.2（WRITE）`compute_region_weights(worldpop, cfg) -> dict`

实现区域需求权重计算（对应技术大纲 §4.1.2）：

- `w_g = pop_g / Σ pop_g`（全局归一化，和为 1）；
- 同时计算每个像元的近似 UTC 偏移 `utc_offset = lon / 15`（用 transform 推算每列经度）；
- **只保留有效像元（pop>0 且非 nodata）**，存稀疏三元组，**不存稠密 (H,W) 数组**——稠密数组 43200×17280×4B≈2.98GB 且大量 0，违反全局工程约定（完整工作流.md §4）。返回：
  ```python
  {
    "valid_rows": np.ndarray,   # (K,) 有效像元行索引，int32
    "valid_cols": np.ndarray,   # (K,) 有效像元列索引，int32
    "weights": np.ndarray,      # (K,) 归一化权重，float32，和为 1
    "lon": np.ndarray,          # (K,) 有效像元中心经度
    "lat": np.ndarray,          # (K,) 有效像元中心纬度
    "utc_offset": np.ndarray,   # (K,) 每像元 UTC 偏移（小时）= lon/15
    "total_pop": float,
    "shape": (H, W),            # 原栅格形状，供 P2/P3 反查
    "transform": tuple,         # rasterio transform，供反查
  }
  ```
  K ≈ 5130 万有效像元，三元组 float32+int32×2 ≈ 0.8GB，可接受。

**注意：**
- **不引入 tz_world shapefile**，用经度/15 近似（技术大纲 §4.1.2 明确）；
- 昼夜调制函数本身在 P3 实现（`D_g(t)`），P1 只产出权重和 UTC 偏移；
- 全程**单进程**读取（rasterio 单线程读即可，避免多进程全局 numpy 状态问题，见全局工程约定）。
