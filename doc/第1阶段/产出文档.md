# 第 1 阶段产出文档：外部数据接入与预处理

## 文档定位

登记 P1 阶段**实际产出**的全部交付物与数据特性，供 P2–P3 合成外生 OD 需求使用。所有数值基于 2026-07-07 真实执行（seed=0，config_hash `849c7c91aa27`）。

---

## 1. 代码模块（`src/data/`）

| 模块 | 关键接口 | 用途 | 使用阶段 |
|---|---|---|---|
| `sndlib_parser.py` | `parse_sndlib_native(text)` | 解析单个 SNDlib native 文件 | P1 |
| `load_traffic.py` | `load_traffic_matrix_archive(cfg, which)` / `detect_time_gaps(times, gran)` | 批量加载 tgz → (T,N,N) 矩阵 + 时间缺口检测 | P1, P3 |
| `time_patterns.py` | `extract_global_intensity(matrix)` / `extract_od_relative_pattern(matrix)` / `winsorize_intensity(S, percentile)` | S(t)、p_od(t)、异常截断 | P1, P3 |
| `resample.py` | `resample_to_timeslot(S, times, target_min)` | 线性插值重采样到 Δt=3min | P1, P3 |
| `worldpop.py` | `load_worldpop(cfg)` / `compute_region_weights(wp, cfg)` | 栅格读取 + 稀疏有效像元权重 | P1, P3 |
| `diagnostics.py` | `plot_s_timeseries` / `plot_diurnal_boxplot` / `plot_autocorr` / `plot_resample_spectrum` | 数据诊断图（P6 复用） | P1, P6 |
| `io.py` | `save_npz` / `load_npz` / `save_pickle` / `load_pickle` | 内部格式读写 | P1–P15 |

入口：`src/p1_main.py`，由 `./scripts/run.sh P1 <seed>` 调用。测试：`tests/test_sndlib_parser.py`（4 测）。

## 3. 数据特性（真实执行结果，P2/P3 必读）

### 3.1 GÉANT（主时间模式源）

| 项 | 值 |
|---|---|
| 实际矩阵数 | **10773**（标称 11460，缺 687 因原始数据缺口） |
| 节点数 | 22 |
| 有向需求 | **441/462**（稀疏，21 对 OD 无流量，矩阵中填 0） |
| 粒度 / 单位 | 15min / Mbps |
| 时间范围 | 2005-05-04 15:30 ~ 2005-08-31 07:45（约 4 个月） |
| 时间缺口 | 6 处，最大 4 天多（6/28→7/3） |
| S(t) 清洗 | winsorize 99.9 分位，11 个尖峰截断，max 4.7亿→375065 Mbps |
| S(t) 中位 | 41635 Mbps（清洗前后一致，说明只削尖峰） |
| 重采样 | 10773 → 56966（3min） |

**关键提示给 P3**：用 `S_clean`（非 `S_original`）驱动总量；GÉANT 的 OD 结构是稀疏的，p_od(t) 部分位置恒 0，P3 合成卫星 OD 时空间结构由 WorldPop 重定义，不直接用 GÉANT 的 OD 拓扑。

### 3.2 Abilene（验证数据集）

| 项 | 值 |
|---|---|
| 实际矩阵数 | 48096（与标称一致） |
| 节点数 | 12 |
| 有向需求 | 131/132（稀疏，1 对无流量） |
| 粒度 / 单位 | 5min / Mbps |
| 时间范围 | 2004-03-01 ~ 2004-09-10（6 个月） |
| 时间缺口 | 4 处，**最大 18 天**（3/14→4/2，已知维护停机） |
| S(t) 清洗 | winsorize 99.9 分位，49 点截断，max 11889→10012 |
| 重采样 | 48096 → 93119（3min） |

**关键提示给 P6**：Abilene 18 天长缺口会影响周期性验证，P6 分析需排除缺口段或分段统计。

### 3.3 WorldPop（地理权重）

| 项 | 值 |
|---|---|
| 栅格 | 43200×17280，EPSG:4326，1km |
| 有效像元 K | 51,241,958 |
| 总人口 | 8,195,439,314（81.95 亿，符合 2025 全球） |
| 权重和 | 1.000000（归一化） |
| UTC 偏移范围 | [-12, 12]（经度/15 近似，无 shapefile） |
| 存储格式 | 稀疏三元组（rows/cols/weights/lon/lat/utc_offset），约 0.8GB |

**关键提示给 P3**：昼夜调制 `D_g(t)` 在 P3 实现，用 `utc_offset` 字段；空间聚合到接入卫星在 P2 覆盖映射后进行。

## 5. 给 P2/P3 的接口契约

P2/P3 通过以下方式复用 P1 产出：

1. **时间模式**：`io.load_npz(data_interim/'geant'/'S_clean.npz')['S_clean']` + `load_pickle(.../'resampled_times.pkl')` 得到 3min 粒度的 S(t) 与时刻；
2. **地理权重**：`io.load_npz(data_interim/'worldpop'/'weights.npz')` 得到稀疏三元组 + UTC 偏移；
3. **节点经纬度**：`load_pickle(.../geant/meta.pkl)['node_coords']`（P3 地理映射验证备用）；
4. **诊断图**：`data/interim/diagnostics/*.png` 供 P6 直接引用；
5. **数据字典**：`DATA_DICT.yaml` 是 P6 数据可信度审计的数据来源。

**P1 不交付**：外生 OD 需求合成（P3）、卫星覆盖映射（P2）、总量标定 k0（P3）。P2/P3 不得改动 P1 产出文件，只能读取。
