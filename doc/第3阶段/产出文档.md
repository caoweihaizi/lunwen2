## 文档定位

登记 P3 阶段**实际产出**的全部交付物与 OD 模型特性，供 P4 仿真器使用。所有数值基于 2026-07-07 真实执行（seed=0）。

## 3. OD 模型特性（真实执行结果）

### 3.1 混合 OD 模型（P3 决策）

| 股 | 比例 | 机制 | 实际比例 |
|---|---|---|---|
| 随机 | 0.4 | 按桶权重独立抽样目的 | 0.400 |
| 重力 | 0.4 | `D ∝ W_d / dist^γ`，γ=1.5，地面大圆距离 | 0.400 |
| 热点 | 0.2 | 10 对地理分散热点走廊，全网 π_hot 均分 | 0.200 |

- **三股严格守恒**：commodity 总量 = base_demand 总量 × k0（守恒比 1.0000）；
- **热点对**：10 对覆盖东亚/中东/欧洲/非洲/北美东/南美/大洋洲/南亚 8 区域；
- **97 个 UTC 桶**（15min 粒度），桶权重的和 = 1.0。

### 3.2 时间划分

| 段 | 索引 | 时隙数 | 比例 |
|---|---|---|---|
| train | [0, 34180) | 34180 | 60.0% |
| val | [34180, 42724) | 8544 | 15.0% |
| calib | [42724, 48421) | 5697 | 10.0% |
| test | [48421, 56966) | 8545 | 15.0% |

### 3.3 总量标定 k0

| 项 | 值 |
|---|---|
| k0 | **1.3554**（P3 回填 config） |
| 标定方法 | 最短路代理分流，二分至 P95≈0.8 |
| 利用率中位 | 0.040 |
| 利用率 P95 | 0.800 ✓ |
| 利用率 max | 3.281 |
| 标定采样 | 200 时隙 |

**关键说明（诚实记录）**：最短路代理下流量分布严重右偏——少数热点链路过载（max 328%），多数链路空闲（中位 4%）。这是因为 66 星网络的热点走廊（东亚-欧洲等）必经少数链路。技术大纲 §4.1.4 的"中位≈0.5"在最短路下不可达。P3 标定到 P95≈0.8（让前 5% 链路接近拥塞阈值），max>1 的热点链路是真实拥塞点，**P4 的 ECMP/MAPPO 路由会均衡负载、refine k0**。这是 P3 代理标定的固有局限，已在 k0.json 与 meta.json 注明。

### 3.4 突发场景

- baseline：无干预；
- burst：3 个源卫星在中间 20% 时段（[40%, 60%)）需求 ×4；
- compound：burst + P4 故障（P4 注入故障侧）。

## 5. 给 P4 的接口契约

P4（离散事件仿真器与离线策略）将使用 P3 产出：

1. **commodity**：`np.load('demand/commodity_baseline.npz', allow_pickle=True)['commodity'][t]` → `(M_t,4)` [src_sat,dst_sat,vol_mbps,flow_type]，这是 P4 仿真器的**外生需求输入**；
2. **场景**：baseline（正常）/ burst（突发）/ compound（突发+故障，故障 P4 注入）；
3. **时间划分**：`split.pkl` 的 train/val/calib/test 索引，P4/P7 窗口生成须在各段内；
4. **k0**：已应用进 commodity，P4 直接用；若 P4 真实路由下利用率偏差大，可 refine k0 并重生成 commodity；
5. **拓扑**：P4 从 P2 `topo_series.npz` 取边表重建邻接，注入故障形成运行拓扑。

**P3 不交付**：故障注入（P4/P5）、路由策略（P4）、链路流量/队列/丢包 ground truth（P4）、遥测缺失（P5）。P3 的 commodity 是"喂给路由的需求"，链路负载由 P4 路由决定（内生）。
