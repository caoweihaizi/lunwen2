# 完整工作流：MUCAR 实验路线

## 0. 文档定位

本文档是 MUCAR 硕士论文的**执行路线图**，把 `技术大纲.md` 的研究内容拆解为按依赖顺序排列的阶段（Phase）。每个阶段是一个可独立验收的里程碑，配有目标、输入、产出、依赖、验收门控和对应大纲章节。

**使用方式：**

- 严格按阶段顺序推进，前一阶段未通过验收门控不得进入后续阶段（数据层尤其重要，错误地基上训练的模型无法溯源）；
- 每阶段"具体要做什么"在本轮先保持框架级粒度，进入该阶段时再展开为子任务清单；
- 用第 5 节的进度勾选表追踪状态；每完成一阶段，在对应章节回填实际参数与结果。

本文档不重复技术细节，技术原理与公式见 `技术大纲.md`；本文档只回答"先做什么、后做什么、何时算完成"。

                              └─────────────────┴─────────────────┘
                                              ↓
                                    P15 统计汇总与论文成稿

**核心依赖逻辑：**

- 外生数据（P1）和拓扑（P2）共同决定 OD 需求（P3）；
- OD 需求经仿真器与离线策略（P4）生成链路流量 ground truth；
- 缺失观测（P5）只在 ground truth 副本上注入；
- P6 是**硬门控**：数据不可信则预测/路由全部无意义，必须回退修复；
- 预测器（P7）多策略预训练后冻结，供 P8/P9 评价和 P10/P11 路由使用；
- 校准区间（P9）是路由置信度门控的输入，故 P11 依赖 P9；
- P12/P13/P14 可在 P11 后并行展开。

## 2. 阶段定义

### Phase 0 — 基础设施与工程约定

- **目标：** 搭好可复现的工程底座，后续所有阶段直接复用。
- **输入：** 无。
- **产出：**
  - 项目目录结构（`data/ configs/ src/ models/ results/ logs/`，数据/代码/结果三分离）；
  - Python 环境 + 依赖锁定（PyTorch/MPS、numpy、rasterio、网络/仿真库）；
  - 配置管理（YAML/Hydra，所有可调参数集中）；
  - 全局随机种子管理（数据生成、模型初始化、路由训练各独立种子链）；
  - 实验追踪（每次运行记录配置哈希、种子、git/代码版本、结果路径）。
- **依赖：** 无。
- **验收门控：** 一条 `make reproduce SEED=0` 能从空结果目录跑出可记录的 dummy 实验。
- **对应章节：** 7.1。

### Phase 1 — 外部数据接入与预处理

- **目标：** 把三个外部数据集读成统一的、可追溯的内部格式。
- **输入：**
  - GÉANT：`directed-geant-uhlig-15min-over-4months-ALL-native.tgz`（11460 矩阵，15min，Mbps）；
  - Abilene：`directed-abilene-zhang-5min-over-6months-ALL-native.tgz`（48096 矩阵，5min）；
  - WorldPop：`global_pop_2025_CN_1km_R2025A_v1.tif`（43200×17280，EPSG:4326，1km）。
- **产出：**
  - GÉANT/Abilene native 矩阵解析器 → 统一时间索引的矩阵数组 + 元数据；
  - 全局总强度时序 \(S(t)\) 与 OD 相对模式 \(p_{od}(t)\) 提取；
  - 重采样到 \(\Delta t=3\) min 的时序；
  - WorldPop 栅格读取 → 区域人口权重 \(w_g\)，昼夜调制函数；
  - 数据字典：文件名、版本、下载日期、SHA256、量纲、时间范围。
- **依赖：** P0。
- **验收门控：** 矩阵数与官方一致（GÉANT 11460、Abilene 48096）；\(S(t)\) 可视化呈现清晰日/周周期；WorldPop 权重和为 1。
- **对应章节：** 4.1.1 / 4.1.2 / 4.7。

---

### Phase 2 — 星座拓扑与覆盖生成

- **目标：** 生成 66 星 Walker 星座的动态拓扑与覆盖关系。
- **输入：** 轨道参数（Walker 配置，固定 66 星）。
- **产出：**
  - 每时隙卫星位置；
  - 每时隙计划拓扑 \(G_t^{\mathrm{orb}}\)（邻接矩阵 + 链路距离/传播时延，四邻居）；
  - 卫星—地面区域覆盖映射（哪些栅格区域由哪颗卫星接入）；
  - 每个计划拓扑快照上的目的势函数 \(\phi_t^{\mathrm{orb}}(i,d)\) 预计算。
- **依赖：** P0。
- **验收门控：** 邻接矩阵每行度数 ≤4 且符合轨内/跨轨结构；覆盖无遗漏无重叠（地面全栅格均被某卫星覆盖）；拓扑周期性与轨道周期一致。
- **对应章节：** 3.1 / 4.1.3 / 6.3。

---

### Phase 3 — 外生 OD 需求构建与标定

- **目标：** 合成策略无关的外生地面 OD 需求，并完成总量↔容量标定。
- **输入：** P1 的时间模式 \(S(t), p_{od}(t)\)、WorldPop 权重 \(w_g\)、昼夜调制；P2 的覆盖映射。
- **产出：**
  - 基础地面区域 OD 需求（时间模式 × 空间权重 × 昼夜调制）；
  - 突发/持续热点/缓慢漂移注入 → 场景 OD 需求（干预在路由前完成）；
  - 区域 OD → 接入卫星 → 按目的卫星聚合的 commodity；
  - **总量放缩标定**：调 \(k_0\) 使映射后链路利用率中位 ≈0.5、P95 ≈0.8，高峰链路可触拥塞阈值。
- **依赖：** P1, P2。
- **验收门控：** 需求总量与 \(S(t)\) 守恒；标定后利用率直方图落在目标区间；突发先于路由生效。
- **对应章节：** 4.2 步骤 3–6 / 4.1.4 / 4.5.1。

---

### Phase 4 — 离散事件仿真器与离线策略

- **目标：** 实现流级逐跳仿真器，跑三类离线行为策略，生成链路级 ground truth。
- **输入：** P3 场景 OD 需求；P2 计划拓扑；链路故障注入（10% 随机关闭 → 运行拓扑 \(G_t^{\mathrm{op}}\)）。
- **产出：**
  - 仿真器核心：commodity 聚合、逐跳分配、链路服务、队列更新（离散时间）、时延与丢包计算；
  - 三类离线策略：Dijkstra、ECMP、Queue-aware stochastic routing；
  - 链路故障注入器（运行拓扑生成）；
  - 四张逻辑表：场景 OD 表、节点—commodity 路由表、链路—commodity 明细表、链路状态真值表；
  - 多策略轨迹合并的离线预训练数据集。
- **依赖：** P2, P3。
- **验收门控：** `carried_load = Σ commodity_load` 守恒；不同策略对同一 OD 产生不同但守恒的链路分布；故障链路不承载数据；队列不超缓存上限。
- **对应章节：** 3.2 / 3.3 / 3.5 / 4.2 步骤 7–11 / 4.3。

---

### Phase 5 — 遥测缺失观测生成

- **目标：** 在完整 ground truth 副本上注入结构化缺失，生成运行时可见观测。
- **输入：** P4 的完整链路状态真值表。
- **产出：**
  - MCAR（20%/40%）、MAR（拥塞相关，总缺失率 20%）、Block Missing（单链路连续 10 时隙）三类缺失观测；
  - 观测掩码 \(M\)、信息年龄 \(\Delta\)、缺失类型标记、到达时间；
  - **关键约束：** 缺失只改变观测，不改变 ground truth；缺失在标签生成后注入。
- **依赖：** P4。
- **验收门控：** 实际缺失率与目标一致；MAR 缺失与利用率正相关；Block 连续长度符合设定；ground truth 未被改动。
- **对应章节：** 4.4 / 4.2 步骤 12。

---

### Phase 6 — 数据可信度验证（实验一）★ 硬门控 ★

- **目标：** 证明生成数据包含后续方法所需的周期、突发、空间异质性与拥塞现象，且因果顺序正确。**这是整个研究的地基，未通过不得进入模型阶段。**
- **输入：** P1–P5 全部产出。
- **产出：** 实验一报告：时间分布与日周期、自相关、空间相关性、跨策略链路流量分布差异、OD→明细→总量可追溯一致性、三类干预执行顺序审计、各策略路径长度/负载分散度/样本占比、不同负载下队列与丢包、各缺失机制统计特征。
- **依赖：** P1, P2, P3, P4, P5。
- **验收门控（全部通过才放行）：**
  - 流量总量与 OD 需求守恒；
  - 四张表主键/外键关系完整，`carried_load` 可由明细聚合还原；
  - 突发先于路由、故障先于标签、缺失只改观测三类顺序审计全过；
  - 不同策略产生不同但守恒的链路分布，无单一策略主导；
  - 周期性与空间相关性可观测；
  - 固定种子可完全复现。
- **对应章节：** 4.7 / 7.2 / 附录 A 前三行。

---

### Phase 7 — 预测模型实现与训练

- **目标：** 实现 Mask-aware GAT-GRU-D，多策略混合数据一次性离线预训练。
- **输入：** P6 通过的多策略离线数据集（含缺失观测、路由状态、历史/未来拓扑）。
- **产出：**
  - GRU-D 时间衰减恢复模块；
  - 节点 GAT + 有向边联合编码；
  - 边级时序解码器 + 单调分位数头（结构保证 \(0\le\hat y^{05}\le\hat y^{50}\le\hat y^{95}\)）；
  - Pinball Loss 训练流程；
  - `policy_id` 仅作分组/采样，不进前向；
  - 冻结的预训练预测器（供 P8/P9/P10/P11 使用）。
- **依赖：** P6（硬门控）。
- **验收门控：** 模型可训练收敛；单调头 quantile crossing rate = 0；推理可在 M4 Pro 内存内运行。
- **对应章节：** 5.1–5.4。

---

### Phase 8 — 链路流量预测实验（实验二）

- **目标：** 验证预测精度与跨策略泛化能力。
- **输入：** P7 冻结预测器；P6 数据集的划分（含 leave-one-policy-out）。
- **产出：**
  - 点预测基线对比（Historical Average / LSTM / GCN-GRU / 本文中位数）：MAE、RMSE、MAPE、峰值 MAE、多步退化、参数量、推理时间；
  - 跨策略泛化：仅 Dijkstra / +ECMP / +Queue-aware 三种训练设置 × 已见/留一策略测试；
  - 路由状态条件消融（无/下一跳/下一跳+分流/完整历史）；
  - MUCAR 零样本策略测试（不设 `policy_id=MUCAR`）；
  - 分位数 Pinball Loss 与 crossing rate 报告。
- **依赖：** P7。
- **验收门控：** 完整数据下不显著弱于强基线；多策略训练优于单策略；加路由状态后跨策略误差与策略间方差下降；crossing rate = 0。
- **对应章节：** 7.3 / 7.3.1。

---

### Phase 9 — 不确定性校准实验（实验三）

- **目标：** 验证经校准区间的覆盖率与因果合法性。
- **输入：** P7 预测器的原始分位数输出；P6 独立校准集与延迟到达标签。
- **产出：**
  - 原始 QR / 固定 CQR / 延迟标签感知滑动 CQR 三方法对比；
  - 指标：PICP、MPIW、Coverage Gap、Winkler Score（目标 90%）；
  - MAR 场景四类分层报告：Observed-label / Full ground-truth / High-load / MAR-hidden；
  - 因果校准审计：预测发出时刻、目标时刻、真值到达时刻、是否永久缺失的完整记录。
- **依赖：** P7。
- **验收门控：** 主要场景 PICP ∈ [87%, 93%]；不存在使用未到达真值或插补标签计算残差的记录；MAR 四类覆盖率均报告，不以可观测子集替代总体结论。
- **对应章节：** 5.5 / 7.4 / 7.6.1。

---

### Phase 10 — 路由环境与 MAPPO 实现

- **目标：** 实现置信度感知的流级逐跳 MAPPO 路由。
- **输入：** P2 拓扑与势函数；P7 冻结预测器；P9 校准区间接口。
- **产出：**
  - POMDP 建模：观测 = (全局 commodity 特征, 候选邻居边特征集, action mask)；
  - 共享参数 actor（候选槽位补齐为 `edge_features[4,F_edge]`）+ centralized critic；
  - action masking（势函数严格下降 + 故障屏蔽 + 无合法动作则等待）；
  - 置信度门控（区间宽度/信息年龄/校准状态 → 可信度指标 \(c_t^{conf}\)）；
  - 奖励函数（只依赖实际时延/丢包/队列/切换，不直接惩罚预测风险）；
  - 切换冷却/滞回抑制振荡；
  - MAPPO 训练流程（集中训练分布执行）。
- **依赖：** P2, P7, P9（P9 提供校准区间给门控）。
- **验收门控：** 流级环境业务守恒；所有非等待转发势值严格下降；故障无合法邻居时执行等待而非选已屏蔽动作；训练可收敛。
- **对应章节：** 6.1–6.6 / 7.6.2。

---

### Phase 11 — 路由综合性能实验（实验四）

- **目标：** 在公平设置下证明置信度感知路由优于点预测路由。
- **输入：** P10 路由实现；P7 冻结预测器。
- **产出：**
  - 六基线对比：Dijkstra / ECMP / MAPPO(无预测) / Point-MAPPO / Missing-aware Point-MAPPO(无不确定性) / MUCAR；
  - 公平性保证：相同 actor/critic 容量、训练预算、奖励；学习型路由复用同一冻结预测器，仅改变可见输出/缺失信息/置信度接口；
  - 指标：平均/P95/P99 端到端时延、丢包率、吞吐、最大利用率、负载均衡度、路由成功率、路径切换次数、利用率振荡、状态交换量。
- **依赖：** P7, P10（P9 校准已内嵌于 P10）。
- **验收门控：** 正常场景 MUCAR 不显著差于 Point-MAPPO；流级环境审计通过。
- **对应章节：** 7.5 / 7.6.2。

### Phase 12 — 压力测试（实验五）

- **目标：** 检验缺失、突发、故障并存时的端到端鲁棒性。

- **依赖：** P11。
- **验收门控：** 复合压力场景 MUCAR 的 P95 时延与丢包稳定优于 Point-MAPPO；切换次数与利用率振荡在可接受范围。
- **对应章节：** 7.6 / 7.6.1 / 7.6.2。

## 3. 阶段间门控规则

| 门控 | 类型 | 含义 |
|---|---|---|
| P6 → P7 | **硬门控** | 数据不可信则禁止训练模型；不通过必须回 P3/P4/P5 修复 |
| P7 冻结 → P10/P11 | 接口门控 | 路由训练期间预测器不得改动；改变预测器需重跑 P8–P13 |
| P9 → P11 | 接口门控 | 路由置信度门控依赖校准区间；校准方法变更需重跑 P10–P13 |
| 单一数据划分贯穿 | 一致性门控 | P7–P15 所有模型使用同一数据划分与预测窗口（7.1） |
| 同一 OD 跨策略同分区 | 防泄漏门控 | 同一 OD 场景不同策略的链路标签必须在同一时间分区（4.6） |

**回退原则：** 任何阶段发现上游问题，修复上游后标记受影响下游阶段为"需重跑"，不得在下游打补丁掩盖。

## 4. 全局工程约定

- **目录：** `data/`（原始+处理，只读）、`configs/`（YAML）、`src/`（代码）、`models/`（ checkpoints）、`results/`（指标+图）、`logs/`（运行日志）。
- **配置驱动：** 所有可调参数（4.1.4 表中全部）写在 YAML，代码不硬编码；`load_config(cli=True)` 支持命令行 `key=value` override（run.sh 透传）。
- **种子语义：** 一次"5 种子实验"中，第 `s` 次运行令 `seed.data = seed.model = seed.routing = s`，即一个 eval seed 同步驱动全部随机源；`run.sh <stage> <seed>` 同时覆盖三条链。`eval_seeds = {0,1,2,3,4}`。
- **随机数可复现性：** 数据加载、仿真、训练凡涉及多进程/多线程处，**不得依赖全局 `np.random` 状态**，必须用独立 `np.random.Generator` 实例（由 `make_seed_stream` 派生子种子）；否则同一种子在不同并发度下结果不同。全局 `seed_everything` 仅用于单线程确定性场景与初始化。
- **可追溯：** 每个结果文件可由 `(config_hash, seed, code_version)` 唯一复现。`config_hash` **剥离 seed**——它标识配置而非运行实例，同配置的 5 个种子共享同一 hash，便于 P15 按配置归组。跨阶段对账（占位参数 walker/k0 回填会使 hash 变化）用 `(stage, code_version, seed)`。
- **防泄漏：** 标准化参数只由训练集估计；校准只用在决策时刻已到达的真实标签；测试集缺失可更强但不交叉。
- **资源预算：** M4 Pro 24GB 统一内存，优先 MPS 回退 CPU；网络宽度/并行环境数/序列长度按 7.1 控制。大栅格/大数组（如 WorldPop 43200×17280）存稀疏三元组或分块，禁存稠密 0 填充。
- **Python：** 3.11（uv 自管，避免 Homebrew pyexpat 符号问题）。
